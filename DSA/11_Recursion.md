# 🔹 1. Definition

**Recursion** is a programming technique where:

> A function **calls itself** to solve a smaller version of the same problem.

👉 It breaks a big problem into **smaller subproblems**.

---

# 🔹 2. Key Concepts

### ✅ 1. Base Case

* Condition where recursion **stops**
* Prevents infinite loop

### ✅ 2. Recursive Case

* Function calls itself with **smaller input**

# 🔹 4. How Recursion Works (Call Stack)

👉 Each function call is stored in **call stack**

Example:

```js id="hptd6x"
function printNum(n) {
    if (n === 0) return;
    console.log(n);
    printNum(n - 1);
}
printNum(3);
```

### Flow:

```
printNum(3)
 → printNum(2)
   → printNum(1)
     → printNum(0) (STOP)
```

👉 Then it **returns back (unwinds)**

# 🔹 5. Basic JavaScript Examples

---

## ✅ 1. Factorial

```js id="mq2kn0"
function factorial(n) {
    if (n === 0 || n === 1) return 1; // base case
    return n * factorial(n - 1);      // recursive case
}

console.log(factorial(5)); // 120
```

---

## ✅ 2. Fibonacci

```js id="oz1g8v"
function fibonacci(n) {
    if (n <= 1) return n;
    return fibonacci(n - 1) + fibonacci(n - 2);
}
```

---

## ✅ 3. Sum of Numbers

```js id="b2b1y4"
function sum(n) {
    if (n === 0) return 0;
    return n + sum(n - 1);
}
```

# 🔹 6. Practical Implementation (Tree Recursion)

👉 Recursion is heavily used in **trees**

```js id="r4z4va"
function inorder(node) {
    if (node === null) return;

    inorder(node.left);
    console.log(node.data);
    inorder(node.right);
}
```

# 🔹 7. Types of Recursion

| Type           | Description                      |
| -------------- | -------------------------------- |
| Direct         | Function calls itself            |
| Indirect       | Function A calls B, B calls A    |
| Tail Recursion | Last operation is recursive call |
| Head Recursion | Recursive call first             |
| Tree Recursion | Multiple recursive calls         |

---

# 🔹 8. Advantages

✅ Simple code
✅ Reduces complexity
✅ Best for tree/graph problems

---

# 🔹 9. Disadvantages

❌ Uses more memory (stack)
❌ Risk of stack overflow
❌ Sometimes slower than iteration

---

# 🔹 10. Time Complexity

Depends on calls:

* Factorial → O(n)
* Fibonacci → O(2ⁿ) (inefficient)

## ⭐ Q2: Check Palindrome

```js id="nsprcc"
function isPalindrome(str) {
    if (str.length <= 1) return true;

    if (str[0] !== str[str.length - 1]) return false;

    return isPalindrome(str.slice(1, -1));
}
```

## ⭐ Q3: Power (xⁿ)

```js id="v9d2mt"
function power(x, n) {
    if (n === 0) return 1;

    return x * power(x, n - 1);
}
```

## ⭐ Q4: Binary Search (Recursive)

```js id="8tb93g"
function binarySearch(arr, left, right, target) {
    if (left > right) return -1;

    let mid = Math.floor((left + right) / 2);

    if (arr[mid] === target) return mid;

    if (target < arr[mid])
        return binarySearch(arr, left, mid - 1, target);
    else
        return binarySearch(arr, mid + 1, right, target);
}
```

# 🔹 12. When to Use Recursion

👉 Use when:

* Problem can be divided into subproblems
* Tree/Graph traversal
* Backtracking problems

---

# 🔹 13. When NOT to Use

👉 Avoid when:

* Large input (stack overflow risk)
* Iteration is simpler
