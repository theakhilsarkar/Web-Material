## **1Ô∏è‚É£ What is an API?**

**API** stands for **Application Programming Interface** ‚Äî it‚Äôs like a **messenger** between two systems, allowing them to talk to each other.

üí° **Analogy:** Think of an API as a **waiter** in a restaurant:

* You (the client) tell the waiter (API) what you want.
* The waiter tells the kitchen (server) your order.
* The kitchen prepares the food and sends it back via the waiter.

---

## **2Ô∏è‚É£ Why APIs Are Important in Development**

* **Data Communication**: Get data from a database or another service.
* **Integration**: Connect with payment gateways, weather data, Google Maps, etc.
* **Reusability**: Once built, APIs can be used across apps/platforms.
* **Scalability**: You can change the backend without touching the frontend.

## **3Ô∏è‚É£ API Types in Web Development**

| API Type          | Example                        | Usage                           |
| ----------------- | ------------------------------ | ------------------------------- |

## **4Ô∏è‚É£ HTTP Methods in REST APIs**

| Method     | Purpose             | Example                |
| ---------- | ------------------- | ---------------------- |
| **GET**    | Retrieve data       | Get all products       |
| **POST**   | Create data         | Add a new product      |
| **PUT**    | Update entire data  | Update product details |
| **PATCH**  | Update partial data | Update product price   |
| **DELETE** | Remove data         | Delete a product       |

## **5Ô∏è‚É£ Status Codes You Should Know**

| Code                          | Meaning                |
| ----------------------------- | ---------------------- |
| **200 OK**                    | Request successful     |

| **400 Bad Request**           | Client sent wrong data |

## **6Ô∏è‚É£ Introduction to Postman**

**Postman** is a popular tool to test APIs without writing code.

* ‚úÖ Send API requests
* ‚úÖ Inspect responses
* ‚úÖ Test different methods (GET, POST, etc.)
* ‚úÖ Automate testing with collections

---

## **7Ô∏è‚É£ Postman Practical ‚Äì Step by Step**

### **Step 1 ‚Äì Install Postman**

* Download from [https://www.postman.com](https://www.postman.com)
* Create a free account.

### **Step 2 ‚Äì Make a GET Request**

1. Open Postman ‚Üí Click **New ‚Üí HTTP Request**.
2. Select **GET** from the dropdown.
3. Enter an API URL, e.g.,

   ```
   https://jsonplaceholder.typicode.com/posts
   ```
4. Click **Send** ‚Üí You‚Äôll see the JSON response.

---

### **Step 3 ‚Äì Make a POST Request**

1. Select **POST** method.
2. Enter the API URL:

   ```
   https://jsonplaceholder.typicode.com/posts
   ```
3. Go to **Body ‚Üí raw ‚Üí JSON** and write:

   ```json
   {
     "title": "New Post",
     "body": "Post body content",
     "userId": 1
   }
   ```
4. Click **Send** ‚Üí You‚Äôll see a **201 Created** response.

---

### **Step 4 ‚Äì Save the Request in a Collection**

* Click **Save** ‚Üí Name your collection (e.g., "Lecture 27.1 APIs").
* This helps in reusing requests for later testing.
